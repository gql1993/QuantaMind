## 1. 引言

### 1.1 编写目的

本说明书依据《长三角一体化国家级AI+量子融合产业创新平台建设方案-详细版本》（以下简称《建设方案》），对平台所涉及的**软件系统**进行详细设计，为开发、测试、部署与运维提供技术依据。读者对象为架构师、开发负责人、各子平台技术负责人及项目管理办公室（PMO）。

### 1.2 范围

- **纳入范围**：以“1个智能中台+6大业务平台”（1+6）为边界的全部软件系统，包括：
  - 量智大脑（量子科技自主科研人工智能中台）及其客户端；
  - 启元、格物、墨子、悬镜、天元、开物六大业务平台中与中台交互、与研发/制造/应用相关的软件模块；
  - 数据中台、知识图谱、统一API与数据标准。
- **不纳入范围**：硬件设备选型、土建与机房、非软件类研发任务（如材料合成、流片工艺），仅在其与软件接口处做说明。

## 2. 系统概述

### 2.1 建设方案与软件的对应关系

《建设方案》明确创新平台采用**“1+6”一体化总体架构**：以**量子科技自主科研人工智能中台（量智大脑）**为智能底座和统一入口，通过标准化接口为六大业务平台提供AI赋能，并接收业务平台反馈数据，形成数据驱动的自我演进闭环。六大业务平台覆盖量子计算“设计—制造—测控—应用”全生命周期。

从软件视角的映射如下：

| 建设方案任务 | 软件系统定位 | 核心软件能力 |
|-------------|-------------|---------------|
| 任务一：量智大脑 | AI中台 + 客户端 | 大模型服务、智能体路由、多Agent协作、知识图谱与数据中台、统一API |
| 任务二：启元 | 全栈软件与Q-EDA研发中心 | 编程/编译栈、Q-EDA工具链、GPU-QPU异构计算软件、与中台“AI芯片设计师”集成 |
| 任务三：格物 | 材料与器件研发中心 | 材料数据库、高通量计算平台、工艺仿真与数据采集、与中台“AI材料科学家”集成 |
| 任务四：墨子 | 数字化制造中心 | MES、数字孪生、智能运维、具身智能调度、与中台“AI工艺工程师/设备运维员”集成 |
| 任务五：悬镜 | 计量与测控中心 | 测控系统软件、自动化校准、基准测试与评测、与中台“AI设备运维员”集成 |
| 任务六：天元 | 云服务与应用平台 | 云平台、任务调度、行业SDK、与中台“自然语言到量子线路”等能力集成 |
| 任务七：开物 | 量子赋能AI基础设施 | 量子机器学习算法库、量超融合编程环境、与天元/中台的数据与任务闭环 |

### 2.2 发展目标在软件上的分解

- **近期（到2027年底）**：中台与各平台**基础软件与接口就绪**；AI中台具备会话、智能体调度与基础工具调用；至少1个业务平台（如启元或天元）与中台完成联调；数据中台与知识图谱V1.0可用。
- **中期（到2029年底）**：中台与六大平台**API级深度融合**；全系列AI智能体通过统一API对外服务；平台间数据流、工作流打通；国产化替代相关软件（如Q-EDA、测控）达到可交付版本。
- **远期（到2030年底）**：**自主科研AI系统**成为平台科研主流入口；云平台与行业解决方案规模化；量超融合与“量子赋能AI”在软件层面形成可演示、可度量的闭环。

## 3. 总体架构设计

### 3.1 软件逻辑架构

```
                    ┌─────────────────────────────────────────────────────────┐
                    │                    用 户 与 应 用 层                     │
                    │  桌面客户端 │ Web客户端 │ 嵌入式面板 │ 行业应用/云用户    │
                    └───────────────────────────┬─────────────────────────────┘
                                                │ HTTPS / WSS / 内网
                    ┌───────────────────────────▼─────────────────────────────┐
                    │              量 智 大 脑（AI 中 台）                      │
                    │  网关 │ 会话 │ 智能体路由 │ 多模型推理 │ 技能/工具执行      │
                    │  知识图谱与数据中台 │ 心跳与自主任务调度                   │
                    └───────────────────────────┬─────────────────────────────┘
                                                │ 标准化 API / 事件 / 数据回写
        ┌───────────┬───────────┬───────────┬───┴───┬───────────┬───────────┐
        ▼           ▼           ▼           ▼       ▼           ▼           ▼
   ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
   │  启元   │ │  格物   │ │  墨子   │ │  悬镜   │ │  天元   │ │  开物   │
   │ 软件栈  │ │ 材料/   │ │ MES/    │ │ 测控/   │ │ 云平台  │ │ QML/    │
   │ Q-EDA   │ │ 器件    │ │ 数字孪生│ │ 校准    │ │ 应用    │ │ 量超融合│
   └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘
        六大业务平台（各平台内部含业务系统、数据库、与中台对接模块）
```

### 3.2 部署架构原则

- **量智大脑**：集中部署于创新平台AI算力中心，对外提供统一域名/内网地址；支持水平扩展（网关与无状态推理节点）。
- **六大平台**：各平台软件可独立部署在本平台机房或共享数据中心，通过**统一API网关或中台侧适配器**与量智大脑通信；敏感数据可保留在平台侧，仅向中台提供脱敏或任务级数据。
- **客户端**：支持桌面（Electron/本地安装）、Web（浏览器）、嵌入式（Q-EDA等工具内嵌），均只与量智大脑网关建立会话，不直连业务平台。

### 3.3 技术选型原则

- **自主可控优先**：操作系统、运行时、基础中间件优先选用国产或开源可控方案；大模型支持国产（通义、DeepSeek等）与开源（如 Llama）组合。
- **统一接口**：中台与各平台之间采用 REST/WebSocket 与统一数据模型（见第7章）；内部可选用 gRPC/消息队列，但对外暴露标准化 API。
- **可观测性**：全链路日志、指标与追踪（如 OpenTelemetry 兼容），满足PMO与运维监控需求。

## 4. 量智大脑（AI中台）详细设计

### 4.1 定位与边界

量智大脑对应《建设方案》**任务一**，定位为整个创新平台的“操作系统”和“智能内核”，在软件上实现三层能力：

1. **赋能工具层**：AI辅助设计、仿真、数据分析等工具与服务，通过统一API与技能（Skills）对外提供。
2. **协同伙伴层**：多智能体（Agent）与人类协同——任务分解、实验方案生成、平台资源调用、结果汇总与报告。
3. **自主发现层**：在特定领域由AI自主提出假设、生成实验、调用工具、分析数据并形成结论，支持闭环验证与知识沉淀。

### 4.2 子系统划分

| 子系统 | 职责 | 与建设方案对应 |
|--------|------|----------------|
| **网关（Gateway）** | 会话管理、认证鉴权、请求路由、流式输出、与客户端协议适配 | 统一入口 |
| **智能体路由（Brain/Agent Router）** | 意图识别、任务分解、专家模型/Agent 选择与编排 | “智能体路由”架构 |
| **多模型推理（Brain）** | 量子物理/材料/工艺/视觉等专用大模型及通用LLM的调用与调度 | 量子科学垂直领域大模型 |
| **工具执行层（Hands）** | 调用启元、格物、墨子、悬镜、天元、开物等平台能力及内部工具 | 全生命周期自主科研 |
| **记忆与知识（Memory）** | 会话记忆、项目上下文、与知识图谱/数据中台查询 | 知识图谱与数据中台 |
| **心跳与自主任务（Heartbeat）** | 定时/事件驱动的自主任务调度、状态上报、异常处理 | 自主科研闭环 |
| **技能市场（Skills）** | 技能注册、版本管理、与工具执行层绑定 | 可扩展能力封装 |

### 4.3 智能体（Agent）与建设方案对应

| 建设方案智能体 | 软件实现要点 |
|----------------|-------------|
| AI理论物理学家 | 专用模型/提示 + 算法与纠错编码知识库；输出为算法描述、伪代码或与启元兼容的电路表示 |
| AI材料科学家 | 材料大模型 + 格物平台数据与高通量计算接口；输出为材料配方、结构建议、与格物对接的任务单 |
| AI芯片设计师 | 与Q-EDA深度集成；版图约束、布线规则、与启元平台API互操作；支持“设计—仿真—反馈”闭环 |
| AI工艺工程师 | 与墨子MES/数字孪生接口；工艺参数建议、良率预测、异常根因分析 |
| AI设备运维员 | 与悬镜/墨子设备数据接口；故障预测、校准建议、维护工单生成 |

上述智能体在软件上可实现为：**单一编排Agent + 多专家模型/子Agent**，由智能体路由按任务类型调度。

### 4.4 客户端形态（与建设方案一致）

- **桌面客户端**：主形态，可安装、支持内网/离线场景，集成对话、任务、知识检索、心跳状态等。
- **Web 客户端**：轻量查看、审批、看板与报表。
- **嵌入式面板**：在 Q-EDA 等工具内嵌，设计时即调起中台能力。
- **CLI**：脚本与自动化、CI 集成。

客户端仅与量智大脑网关通信，不直接访问六大平台；所有对平台的调用由中台“工具执行层”完成。

### 4.5 与现有 QuantaMind 的对应关系

若采用基于 OpenClaw 的 QuantaMind 实现量智大脑，则：

- **Gateway** ↔ 创新平台统一入口与会话管理；
- **Brain** ↔ 智能体路由 + 多模型推理；
- **Hands** ↔ 六大平台工具封装与执行；
- **Memory** ↔ 记忆与知识图谱/数据中台查询；
- **Heartbeat** ↔ 自主任务与心跳；
- **Skills** ↔ 量子科研技能与平台能力封装。

后续优化可在不改变“1+6”边界的前提下，对 QuantaMind 各模块做功能增强与性能优化。

## 5. 六大业务平台软件设计

### 5.1 启元平台（量电融合全栈与Q-EDA）

**建设方案要点**：自主量子计算编程与编译软件栈、AI驱动的Q-EDA工具链、GPU-QPU异构计算系统软件。

**软件模块设计**：

- **编程与编译栈**：语言前端、优化器、多后端（超导/离子阱/光量子等）、与运行时的统一接口；提供 REST/本地 API 供中台“AI芯片设计师”调用（如提交电路、获取优化结果）。
- **Q-EDA 工具链**：版图设计、布线、微波完整性仿真、与工艺设计套件（PDK）集成；提供“设计—导出—仿真”API 及可选嵌入式 QuantaMind 面板入口。
- **异构计算软件**：任务调度器、CPU/GPU/QPU 资源抽象、与天元/开物平台的作业提交接口；支持中台下发的“混合任务”描述并返回结果。

**与中台接口**：接收中台下发的设计任务、电路/参数查询；向中台回写仿真结果、优化建议、错误日志（脱敏后可用于模型微调）。

### 5.2 格物平台（量子材料与器件）

**建设方案要点**：AI驱动材料设计、高通量计算与表征数据、低温器件（Cryo-CMOS）设计仿真。

**软件模块设计**：

- **材料数据库与高通量计算平台**：材料库、表征数据存储、计算任务队列；向中台开放“材料筛选/设计请求”与“结果查询”API。
- **工艺与器件仿真**：与制造、测试数据对接；提供器件性能预测、工艺窗口分析接口。
- **数据采集与标准化**：与墨子/悬镜的工艺与测试数据对接，统一数据模型与编码。

**与中台接口**：中台“AI材料科学家”提交候选材料或参数；格物返回计算/实验状态与结果，支撑中台闭环优化。

### 5.3 墨子平台（量子数字化制造）

**建设方案要点**：黑灯工厂、数字孪生、MES、智能运维、具身智能调度。

**软件模块设计**：

- **MES**：工单、工艺路线、设备绑定、质量追溯；向中台提供生产状态、良率汇总、异常事件（可配置脱敏）。
- **数字孪生**：产线模型、仿真执行、与物理产线数据同步；支持中台“AI工艺工程师”的工艺建议在孪生环境中先仿真再下发。
- **智能运维与具身调度**：设备健康、故障预测、维护工单；机器人/具身任务由统一调度服务执行，中台可下发“建议操作”或“审批后执行”。

**与中台接口**：中台查询生产/设备状态、提交工艺参数建议或维护建议；墨子回写执行结果与关键指标，用于中台模型与策略迭代。

### 5.4 悬镜平台（量子计量与测控）

**建设方案要点**：万比特级测控系统、AI驱动一键校准、基准测试（Benchmarking）体系。

**软件模块设计**：

- **测控系统软件**：波形与参数配置、数据采集、实时反馈；提供“校准任务”“基准测试任务”的 API。
- **自动化校准服务**：与中台“AI设备运维员”对接；接收校准目标与约束，返回校准方案或执行结果。
- **基准测试与评测**：测试集管理、指标计算、报告生成；对外提供标准化的性能指标 API，供天元/中台展示与对比。

**与中台接口**：中台发起校准或评测任务；悬镜返回状态与报告，中台可据此生成自然语言总结或建议。

### 5.5 天元平台（量子计算云服务与应用）

**建设方案要点**：量超融合云平台、行业解决方案SDK、自然语言到量子线路、量子赋能AI验证。

**软件模块设计**：

- **云平台核心**：用户与权限、任务队列、资源调度、计费与配额；支持量子真机与经典算力统一入口。
- **应用与SDK**：行业解决方案封装、SDK/API 文档、示例与模板；与中台“自然语言到量子线路”能力结合，提供统一开发体验。
- **与开物/中台闭环**：为“量子赋能AI”提供算力与任务接口；云上产生的作业与结果数据（经脱敏）可反哺中台与开物算法优化。

**与中台接口**：中台代表用户提交作业、查询状态、获取结果；天元向中台开放资源使用统计与可选的反馈数据，用于智能体与调度优化。

### 5.6 开物平台（量子赋能AI基础设施）

**建设方案要点**：量子机器学习算法库、量超融合编程环境、AI任务驱动的混合计算框架。

**软件模块设计**：

- **量子机器学习库（QML）**：算法实现、与经典框架（如 PyTorch）的接口、与天元/真机后端的对接。
- **量超融合编程环境**：统一编程模型、任务分解与调度、低延迟数据交换；对上层暴露“AI+量子”统一 API。
- **应用示范支撑**：与天元行业方案、中台“量子赋能AI”演示场景对接，提供可度量、可复现的流水线。

**与中台接口**：中台可发起“量子增强AI”类任务（如优化、采样）；开物返回任务状态与结果，支撑中台报告生成与策略迭代。

## 6. 数据中台与知识图谱设计

### 6.1 定位（与建设方案一致）

数据中台是量智大脑的“数据燃料库”和全平台数据枢纽，实现：

- 全生命周期数据归集与治理（设计—制造—测控—应用）；
- 与AI能力中台深度交互：训练数据、知识图谱查询、实时数据流；
- 动态演进的量子知识图谱（文献、专利、平台自产数据）；
- 数据服务与闭环：标准化 API、反馈机制，支撑“数据→AI→洞察→决策→新数据”闭环。

### 6.2 数据域与主题

- **设计域**：电路、版图、仿真配置与结果、Q-EDA 元数据（来自启元）。
- **材料与器件域**：材料配方、表征数据、器件参数（来自格物）。
- **制造域**：工艺参数、设备日志、良率与追溯（来自墨子）。
- **测控域**：校准参数、基准测试结果、设备状态（来自悬镜）。
- **应用域**：作业定义、运行结果、用户行为（来自天元/开物）。
- **知识域**：文献、专利、术语、关系型知识（知识图谱）。

### 6.3 知识图谱

- **实体类型**：材料、器件、工艺、算法、设备、论文、专利、项目、人员等。
- **关系**：工艺-材料、算法-硬件、论文-实体引用等；支持从平台数据中自动抽取与人工校验。
- **接口**：面向中台的图谱查询 API（实体、关系、路径、摘要），支持自然语言问询转图谱查询。

### 6.4 数据安全与合规

- 分级分类：按敏感程度标记；高敏感数据可仅存于平台侧，中台仅接收聚合/脱敏/任务级结果。
- 访问控制：按角色与项目控制；审计日志覆盖数据访问与导出。
- 数据跨境与共享：若涉及国际合作，遵循《建设方案》中“数据安全港”与白名单机制，在软件上支持策略引擎与审计。

## 7. 接口设计

### 7.1 中台统一 API 原则

- **认证**：统一采用令牌（Token）或内网双向认证；客户端与各平台调用中台均需鉴权。
- **协议**：REST（资源型读写）+ WebSocket（流式对话、实时状态）；必要时内部使用 gRPC/消息队列。
- **数据格式**：JSON 为主；大文件或流式数据可约定分片与编码方式。
- **版本**：URL 或 Header 中带 API 版本（如 `/api/v1/`），保证向后兼容与灰度发布。

### 7.2 中台与客户端

- **会话**：`POST /api/v1/sessions` 创建会话；`GET /api/v1/sessions/{id}` 查询。
- **对话**：`POST /api/v1/chat` 支持流式（stream=true）与非流式；请求体含 session_id、message、可选 tool_preference。
- **任务与心跳**：任务列表、状态、日志通过专用接口或 WebSocket 推送；心跳状态可 REST 轮询或 WS 订阅。

### 7.3 中台与六大平台

- **抽象**：各平台在中台侧抽象为“工具”或“技能”；中台通过 Hands 层调用平台提供的 REST/消息接口。
- **统一约定**：
  - 请求：任务类型、参数、优先级、回调地址（可选）。
  - 响应：任务 ID、状态（排队/执行中/成功/失败）、结果或错误码。
  - 异步：长任务采用“提交—轮询/回调”模式，平台提供任务状态与结果查询 API。
- **数据回写**：平台按约定格式向数据中台或中台指定端点推送脱敏/聚合数据，用于训练与知识更新。

### 7.4 平台间（可选）

- 启元—墨子：设计结果与工艺参数传递；
- 墨子—悬镜：制造批次与测控任务关联；
- 天元—开物：云任务与量超融合资源调度。

平台间接口由各平台与 PMO 联合定义，建议优先经中台或数据中台中转，便于审计与统一治理。

## 8. 非功能性设计

### 8.1 安全

- **身份与访问**：统一身份与 RBAC；敏感操作二次确认与审计。
- **通信**：TLS 全链路；内网可叠加证书或 mTLS。
- **数据**：静态数据加密、密钥管理；敏感字段脱敏策略与合规检查。
- **供应链**：依赖组件清单与漏洞扫描，关键组件国产化或可控替代。

### 8.2 性能与可用性

- **中台**：网关与推理节点无状态、可水平扩展；关键路径延迟目标（P99）根据业务细化（如对话首字 < 2s）。
- **平台**：各平台自定 SLA；与中台交互的接口需满足超时与重试约定，避免级联故障。
- **高可用**：中台关键组件多实例、负载均衡；数据库与消息队列按需主从/集群。

### 8.3 可扩展性

- **智能体与技能**：新增 Agent 或 Skill 通过配置与注册接入，无需改动网关核心。
- **平台接入**：新平台或新系统通过“工具/技能”注册与适配器接入中台，符合统一 API 与数据规范即可。
- **模型**：支持新增专用大模型或第三方模型，由智能体路由与配置管理。

### 8.4 可观测性

- **日志**：结构化日志、统一采集与检索。
- **指标**：QPS、延迟、错误率、队列深度；中台与各平台暴露 Prometheus 兼容指标。
- **追踪**：请求 ID 贯穿中台与平台，支持分布式追踪与问题定位。
