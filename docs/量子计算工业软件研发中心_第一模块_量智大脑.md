## 一、建设必要性与平台定位

### （一）行业发展现状与战略必要性

当前，量子信息与人工智能正在从“并行发展”加速走向“深度耦合”。国际上，AI for Science、AI for Quantum、Quantum for AI 已成为主要科技强国和头部企业布局的前沿方向。量子计算研发活动也正在由传统的串行人工研发模式，演进为以数据驱动、模型驱动、智能体驱动为特征的新范式。

从全球趋势看，量子领域正在出现三类重大变化：

1. **AI 正在全面渗透量子研发链条。** 从材料发现、器件设计、版图优化、工艺调参、自动化校准到量子纠错，AI 已不再是单点工具，而是开始承担跨流程的协同优化角色。
2. **异构算力与软件中台成为主流架构。** 未来量子计算不再是单一 QPU 的问题，而是 CPU、GPU、QPU 以及实验设备、工业系统共同参与的复杂系统工程，需要统一的编排与智能中枢。
3. **自主科研 AI 系统开始成为新一代科研基础设施。** 国际前沿已出现 AI Co-Scientist、AI Scientist 等范式探索，表明 AI 正从“辅助工具”向“科研伙伴”甚至“自主研究主体”演进。

与此相对应，我国在量子计算工程化和产业化进程中仍面临以下突出问题：

1. **核心软件体系尚不完整。** 量子计算软件、Q-EDA 工具、自动化测控软件、工艺优化平台、知识管理平台等仍缺乏统一智能底座，软件链条碎片化问题突出。
2. **研发数据难以形成闭环资产。** 设计数据、仿真数据、制造数据、设备数据、测试数据、文献数据长期分散，难以支撑面向全生命周期的学习、优化和决策。
3. **模型能力与专业工具之间存在断层。** 通用大模型难以直接理解量子工程任务，也难以安全调用专业软件与工业系统，导致大模型停留在“问答层”，无法进入“执行层”。
4. **缺乏支撑跨平台协同的智能中台。** 在设计、制造、测控、应用等多环节之间，尚未形成统一入口、统一调度、统一知识底座和统一智能体机制，制约工程化放大与产业化复制。
5. **科研组织方式仍偏传统。** 当前多数研发流程仍依赖专家经验和人工串行协作，尚未形成“AI 参与假设生成、方案设计、工具调用、结果分析、知识沉淀”的自主科研新模式。

因此，建设“量智大脑”不是单纯开发一个对话系统，而是建设面向量子科技全生命周期的**自主科研 AI 中台**。它的战略价值在于：通过构建统一的 AI 能力层和数据-知识-工具闭环，突破我国在量子工业软件体系中的关键短板，推动量子科技研发从“经验驱动、流程割裂”走向“智能驱动、全链协同、持续进化”。

### （二）平台定位

**量智大脑**定位为量子计算工业软件工程技术研发中心的**智能心脏、操作系统和统一能力总线**，是“1 个智能中台 + N 类专业平台”架构中的核心底座。

其定位可概括为四个层次：

1. **统一入口层。** 面向科研人员、工程师、管理者和外部生态伙伴，提供桌面端、Web 端、CLI 端等统一入口，屏蔽底层复杂性。
2. **统一中枢层。** 通过 Gateway、Brain、Orchestrator、Hands、Memory、Heartbeat、Skills 等模块，统一完成会话管理、模型调度、工具调用、知识回注与后台自主运行。
3. **统一赋能层。** 为设计、仿真、制造、测控、知识工程、数据分析、项目管理等领域平台提供可复用的智能体和 API 能力。
4. **自主科研引擎层。** 以量子科学垂直模型、知识图谱、实验数据、外部情报和多智能体协作为基础，逐步实现从 AI 辅助决策向 AI 自主发现的能力跃迁。

**平台核心使命**是：

- 建设面向量子科技的专用 AI 能力体系，而非泛化通用助手。
- 建设面向全生命周期的工具编排体系，而非单点聊天能力。
- 建设面向科研资产沉淀的知识与数据底座，而非孤立的信息检索服务。
- 建设面向未来自主科研的演进框架，而非一次性项目交付。

### （三）现有基础与建设提升方向

量智大脑当前项目已经完成了原型级基础能力搭建，具备向平台化建设演进的现实基础。

| 现有基础 | 当前状态 | 建设提升方向 |
|------|------|------|
| 网关与客户端 | 已具备 FastAPI、WebSocket、CLI/Web/桌面入口基础 | 提升为统一接入、权限治理、审计留痕、可观测的正式平台入口 |
| 大模型接入 | 已具备本地 Ollama 等可插拔模型接入能力 | 升级为量子垂直模型 + 通用模型 + 专家模型的混合专家体系 |
| 智能体路由 | 已实现多角色 Agent 原型与关键词路由 | 升级为面向任务复杂度、上下文和资源状态的动态智能路由 |
| 工具执行层 | 已初步对接设计、仿真、MES、测控、数据、情报等工具能力 | 升级为统一契约、标准 API、权限控制、可编排的工业级工具总线 |
| 记忆与知识 | 已具备项目记忆、资料库、情报摘要等基础能力 | 升级为知识图谱、向量检索、全生命周期数据中台与知识治理体系 |
| 自主运行 | 已有 Heartbeat、定时情报等基础机制 | 升级为自主巡检、自主调优、自主实验建议和自主科研闭环能力 |

换言之，当前 QuantaMind 已经证明“量智大脑”技术路线是**可行的**；本方案的目标是在此基础上进一步完成从**原型验证**到**平台建设**、从**局部可用**到**体系可用**、从**辅助问答**到**自主科研**的升级。

## 二、建设目标与规划

### （一）总体建设目标

面向 2026-2030 年建设期，量智大脑总体目标是：建成国内领先、国际有影响力的量子科技自主科研人工智能中台，形成支撑量子计算“设计-仿真-制造-测控-应用”全生命周期的智能化能力体系，成为量子计算工业软件工程技术研发中心的统一智能底座。

总体目标具体体现在六个方面：

1. **建成统一智能底座。** 完成面向量子科技的 Gateway、模型中枢、智能体路由、工具编排、知识图谱、项目记忆、自主心跳等核心基础设施建设。
2. **建成量子科学专用 AI 能力体系。** 构建量子物理、芯片设计、工艺制造、测控校准、数据分析、知识工程等领域的智能体矩阵。
3. **建成工具与数据双闭环。** 实现设计、仿真、制造、测控、MES、SECS/GEM、数据仓库、云平台等系统的统一接入、统一治理和统一调用。
4. **建成知识驱动平台。** 形成覆盖文献、专利、实验日志、设计文档、制造记录、测试结果的动态知识图谱与数据资产湖。
5. **建成自主科研原型能力。** 在部分典型任务上实现 AI 自主提出假设、分解任务、调用工具、分析结果和生成建议的闭环验证。
6. **形成平台化输出能力。** 通过标准 API、智能体服务和可复制方案包，为中心其他平台以及合作单位提供持续赋能。

### （二）分阶段规划（三年、五年、十年）

#### 1. 三年目标：从原型验证到平台成型

三年内重点完成“从 0 到 1”的能力构建，形成可运行、可集成、可验证的量智大脑平台基座。

重点目标包括：

- 建成量智大脑 V1.0，总体架构稳定运行，支持桌面端、Web 端和 API 统一接入。
- 建成量子科技知识图谱 V1.0 和项目级记忆体系，初步实现文献、项目、实验、工具结果的统一沉淀。
- 发布首批核心智能体，包括 AI 芯片设计师、AI 理论物理学家、AI 工艺工程师、AI 设备运维员、AI 数据分析师、AI 情报员。
- 完成与设计、仿真、测控、MES、数据平台等优先工具集的对接验证，形成不少于 3 个端到端示范场景。
- 在量子材料筛选、版图优化、工艺参数推荐、自动情报摘要等方向完成自主科研概念验证。

#### 2. 五年目标：从平台成型到体系协同

五年内重点完成“从 1 到 N”的体系化融合，形成对研发中心多平台的深度赋能能力。

重点目标包括：

- 建成量智大脑 V2.0，形成混合专家模型路由和跨平台智能体协同体系。
- AI 赋能覆盖中心重点研发制造流程的 50% 以上，支撑设计、制造、测控和应用平台高效协同。
- 形成面向 Q-EDA、数字化制造、量子测控、数据治理、云服务的标准 API 与智能体服务接口。
- 实现 3-5 类关键核心软件或关键环节的国产化替代赋能。
- 推动 AI 参与逻辑比特设计验证、自动化校准和复杂工艺调优，形成具有示范意义的联合场景成果。

#### 3. 十年目标：从体系协同到生态引领

十年内重点实现“从 N 到生态引领”的跨越。

重点目标包括：

- 建成具备国际影响力的量子自主科研 AI 平台。
- 形成面向量子科研 AI 编排、实验数据组织、工具接口协议的标准规范体系。
- 在部分科研方向上实现 AI 自主承担子任务、持续生成高价值科研成果的能力。
- 建成围绕量智大脑的模型、工具、数据、知识、智能体、应用开发生态。

## 三、技术体系架构

### （一）技术体系架构图

**图 1 量智大脑平台技术体系架构图**

量智大脑总体采用“客户端入口层 + 智能中枢层 + 工具与数据层 + 自主演进层”的四层技术体系：

1. **客户端入口层。** 包括桌面客户端、Web 客户端、CLI 客户端和嵌入式面板，是统一的人机交互界面。
2. **智能中枢层。** 由 Gateway、Brain、Orchestrator、Agent Router 构成，负责会话管理、权限控制、模型调度、智能体分发和任务流水线执行。
3. **工具与数据层。** 由 Hands、Knowledge、Memory、State Store、Data Connectors、Sidecar API 构成，负责外部工具调用、数据接入、知识治理和状态持久化。
4. **自主演进层。** 由 Heartbeat、Skills、Feedback Loop、Model Tuning、Policy Engine 构成，负责后台任务、自主巡检、策略优化和能力迭代。

### （二）核心架构说明

**1. Gateway：统一入口与连接器。**  
面向客户端和外部系统提供标准化 REST/WS 接口，负责会话、认证、系统路由、心跳状态和静态资源管理，是量智大脑的统一入口。

**2. Brain：量子科学混合专家推理引擎。**  
不局限于单一通用 LLM，而是面向量子科学任务，形成“通用模型 + 领域模型 + 代码模型 + 专家规则”的协同推理框架。

**3. Orchestrator / Agent Router：任务分发与智能体编排中枢。**  
面向复杂任务，根据任务类型、上下文、资源状态和历史经验，将请求动态路由到合适的智能体及工具链上。

**4. Hands：面向量子科技的工具执行总线。**  
将设计、仿真、制造、测控、数据和情报系统统一封装为可编排工具，形成 AI 能够安全调用的“执行层”。

**5. Knowledge + Memory：知识图谱与项目记忆双底座。**  
前者解决事实、关系、文献和资产组织问题，后者解决项目上下文、决策历史和阶段性认知积累问题。

**6. Heartbeat + Skills：自主运行与能力扩展双引擎。**  
Heartbeat 支撑后台任务、自主巡检、自主提醒；Skills 支撑以 YAML/Markdown/规则包为载体的快速能力扩展。

### （三）数据流与控制流

- **控制流：** 用户请求 → Gateway → Orchestrator → Agent Router → Brain → Hands → 外部系统/数据层。
- **数据流：** 外部系统结果 → 对话上下文 → Memory → Knowledge → 报告/推送/反馈。
- **学习流：** 执行结果、人工反馈、指标变化 → Policy Engine / Tuning → 模型与智能体持续优化。

### （四）与中心其他平台的关系

量智大脑不是独立孤岛，而是研发中心其他平台的共同智能底座。它为 Q-EDA、材料器件、数字制造、测控计量、云平台、量子赋能 AI 等平台提供统一的：

- 智能体能力
- 数据与知识服务
- 工具编排服务
- 自主任务调度能力
- 分析决策与报告生成能力

## 四、重点攻关任务

**任务总述：** 量智大脑设置 **6 项**重点攻关任务，按“基础底座 → 核心智能 → 数据知识 → 自主闭环 → 平台输出”逻辑展开。总体上分为三层：  
第一层是**中台底座**，解决统一入口、统一执行、统一数据问题；  
第二层是**核心智能**，解决模型、路由、智能体与科研任务协同问题；  
第三层是**自主闭环**，解决从辅助决策到自主科研的能力跃迁问题。

---

### 子任务一：建设统一接入与量子科研智能网关

#### 1. 任务背景与任务目标

量子科技场景用户角色复杂、终端形态多样、部署环境以内网和专网为主，必须构建统一、稳定、安全、可治理的接入层。  
本任务目标是建设面向桌面端、Web 端、CLI 端及外部 API 的统一智能网关，形成会话管理、权限治理、任务接入、流式对话和平台可观测能力。

#### 2. 任务架构设计

**图 2 子任务一：Gateway 与客户端接入架构图**

子模块包括：

- 接入网关
- 统一认证与权限控制
- 会话与上下文管理
- WebSocket/流式通道
- 系统健康与监控接口
- 面向其他平台的开放 API

#### 3. 关键技术 / 核心技术创新

**关键技术一：面向科研复杂场景的统一会话总线**  
支持长会话、多轮推理、工具回调、任务中断和状态恢复。

**关键技术二：内网优先与多部署形态适配**  
兼容桌面、浏览器、实验室终端、嵌入式面板和专网环境。

**关键技术三：平台化可观测能力**  
支持健康检查、任务状态、工具调用链、异常追踪和执行审计。

#### 4. 核心指标

| 指标 | 目标值 |
|------|------|
| 核心接口可用率 | ≥99.5% |
| 会话恢复成功率 | ≥95% |
| 流式对话稳定性 | 长会话场景无异常中断 |
| 客户端形态支持 | 桌面/Web/CLI 全覆盖 |

### 子任务二：建设量子科学混合专家模型与智能体路由中枢

#### 1. 任务背景与任务目标

量子科技任务具有高度专业化特征，仅依赖单一通用大模型难以满足设计、理论、制造、测控、数据和知识工程等复杂任务需求。  
本任务目标是构建面向量子科技的混合专家推理引擎与智能路由体系，实现“模型组合 + 智能体组合 + 工具组合”的协同求解。

#### 2. 任务架构设计

**图 3 子任务二：Brain 与 Orchestrator 工具循环示意图**

技术框架包括：

- 通用大模型接入层
- 量子物理、设计、工艺、数据等垂直模型层
- Agent Router 智能路由层
- Tool Call 推理执行循环
- 结果回注与策略优化层

#### 3. 关键技术 / 核心技术创新

**关键技术一：混合专家模型路由**  
针对不同任务，动态调用通用模型、代码模型、领域模型和规则系统，而不是固定模型单点输出。

**关键技术二：科研任务分解与多智能体协同**  
将复杂问题拆解为设计、仿真、分析、总结、执行等子任务，分配给相应智能体协作完成。

**关键技术三：可审计推理链与工具调用链**  
支持从用户问题到工具执行再到结论输出的全过程可追溯。

#### 4. 核心指标

| 指标 | 目标值 |
|------|------|
| 路由准确率 | ≥85%，持续提升 |
| 工具调用成功率 | ≥90% |
| 多智能体协同任务成功率 | ≥80% |
| 关键任务可审计率 | 100% |

## 五、团队建设

### （一）团队建设目标

围绕量智大脑建设，组建一支覆盖 AI、量子科技、工业软件、数据治理、产品平台、安全合规和科研管理的复合型团队，形成“平台架构 + 领域专家 + 工程实现 + 数据治理 + 运行保障”协同体系。

### （二）建议组织架构

建议设置以下团队：

- 平台架构与中间件组
- 模型与智能体组
- 工具集成与工业接口组
- 数据中台与知识工程组
- 客户端与产品体验组
- 测试验证与运维保障组
- 领域专家委员会（设计、材料、制造、测控、应用）
- PMO 与标准化管理组

### （三）保障机制

1. **技术保障机制。** 建立架构评审、接口评审、安全评审、模型评审和里程碑验收机制。  
2. **数据保障机制。** 建立数据采集标准、权限分级、脱敏规则和知识治理流程。  
3. **工程保障机制。** 建立健康检查、回归测试、容量评估、变更审计和发布管理机制。  
4. **协同保障机制。** 通过 PMO 和跨平台协调机制保障量智大脑与其他平台协同推进。  
5. **专家保障机制。** 建立高水平专家咨询委员会，定期评估技术路线与建设成果。

### （四）建设可行性说明

现有 QuantaMind 项目已经完成了中台雏形验证，说明量智大脑具备明确的技术抓手和现实基础；同时，量智大脑采用模块化、可演进、开源可控技术路线，便于在建设期内按照“先原型、再平台、后体系、终闭环”的路径稳步推进，具备较高建设可行性。
