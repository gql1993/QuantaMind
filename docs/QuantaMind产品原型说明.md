## 一、产品定位与用户

### 1.1 产品一句话

QuantaMind 客户端是**量子科研人员与 AI 科学家团队交互的唯一入口**：通过对话、任务与自主发现，完成芯片设计加速、制造-测控闭环与自主科研，背后能力由量智大脑服务端与六大平台提供。

### 1.2 目标用户

| 用户角色 | 主要场景 | 使用形态 |
|----------|----------|----------|
| 量子芯片设计师 | 设计需求澄清、版图/仿真/DRC、设计迭代、导出审批 | 桌面客户端、Q-EDA 嵌入式面板 |
| 工艺/设备工程师 | 工单与良率查看、工艺建议、告警处理、校准与测控任务 | 桌面客户端、Web 看板 |
| 科研负责人 | 自主发现审阅、任务审批、项目与知识检索、周报 | 桌面客户端、Web |
| 算法/仿真工程师 | 与理论物理学家/芯片设计师 Agent 协同、文献与知识检索 | 桌面客户端、CLI |

### 1.3 核心价值

- **一个入口**：对话、任务、审批、自主发现、技能均在同一客户端完成，无需切换多系统。
- **AI 科学家随时在线**：理论物理、材料、芯片设计、工艺、设备运维等 Agent 通过自然语言与技能调用。
- **与平台联动**：设计→制造→测控的数据与任务在客户端可追溯、可审批，数据统一在数据中台。

## 三、关键页面与交互

### 3.1 对话页（主界面）

**布局：**

- 左侧：会话列表（当前项目下的会话，可新建、切换、删除）。
- 右侧上：当前会话的对话流（用户消息 / Agent 回复 / 工具调用与结果折叠块）。
- 右侧下：输入框 + 发送按钮；可选“@ 技能”快捷触发。

**交互要点：**

- 支持流式输出：Agent 回复逐字/逐段出现，可中断。
- 工具调用时展示“正在调用 xxx（Q-EDA / MES / 测控…）”，完成后可展开查看参数与结果。
- 需审批时：对话中插入“待审批：导出 GDS”卡片，用户点击通过/拒绝。

**原型要点：**

- 至少展示 2 轮对话（用户 + Agent 回复）。
- 一条“工具调用”示例（如：调用 Q-EDA 创建设计）。
- 一条“待审批”示例。

### 3.2 任务页

**布局：**

-  Tab：全部 / 我发起的 / 待我审批 / 已完成。
- 列表：任务类型、标题、状态、创建时间、操作（查看/审批）。
- 详情抽屉或右侧滑出：任务参数、执行日志、结果摘要、关联会话。

**交互要点：**

- 待审批：列表内或详情内“通过”“拒绝”按钮。
- 长任务：展示“进行中”，可刷新状态；完成后可查看结果并跳转至相关会话。

**原型要点：**

- 列表含 3–5 条示例任务（含 1 条待审批）。
- 详情展示任务类型（如“Q-EDA 仿真”）、状态、结果摘要。

### 3.3 自主发现页

**布局：**

- 筛选：时间范围、发现类型（文献/实验建议/假设验证/周报）。
- 列表：卡片式，每条含标题、摘要、时间、来源（Heartbeat 层级）；点击可展开或“在对话中继续”。

**交互要点：**

- “在对话中继续”：预填一句 prompt 并跳转到对话页，便于用户深化。

**原型要点：**

- 2–3 条发现卡片（如“文献：xxx”“实验建议：下一组参数”）。

### 3.4 技能页

**布局：**

- 搜索/筛选：按名称、触发词。
- 列表：技能名称、简短描述、触发词、操作“在对话中使用”。

**交互要点：**

- “在对话中使用”：跳转对话页并预填触发词或技能名，发送后由 Brain 匹配技能执行。

**原型要点：**

- 列出 4–6 个示例技能（如 Transmon 设计、良率查询、校准任务）。

### 3.5 设置页

**布局：**

- 分组：连接（Gateway 地址、认证）、外观（主题）、通知、关于。
- 连接：输入框 Gateway URL、API Key（可选）；测试连接按钮。
- 关于：产品名、版本、依赖（如 OpenClaw）说明。

**原型要点：**

- 表单占位 + 测试连接按钮即可。

## 四、用户流程（关键路径）

### 4.1 对话 → 设计加速（主路径）

1. 用户打开客户端，进入“对话”，选择或新建会话（可选绑定项目）。
2. 输入自然语言需求，如“设计一个 5 比特 transmon 芯片，目标频率 5 GHz”。
3. 系统流式回复，并展示“正在调用 Q-EDA 创建设计…”；完成后展示设计 ID 与关键参数摘要。
4. 用户继续追问“跑一下 DRC”，Agent 再次调用工具并展示结果；若有违规则列出，用户可要求迭代。
5. 用户说“导出 GDS”；系统生成待审批，用户在“任务”或对话内审批通过后，Hands 执行导出并返回结果。

### 4.2 自主发现 → 深化（闭环路径）

1. 用户在“自主发现”看到“实验建议：基于昨日 T1/T2 数据，推荐下一组 Rabi 参数…”。
2. 点击“在对话中继续”，跳转对话并预填“请根据这条实验建议生成具体实验方案”。
3. Agent 结合 Memory/数据中台数据回复方案，用户可要求发起测控任务或仅保存到项目记忆。

### 4.3 审批流

1. 对话或任务中出现“待审批：xxx”（如 GDS 导出、制造任务提交）。
2. 用户进入“任务” → “待我审批”，或直接在对话卡片点击“通过/拒绝”。
3. 通过后，任务继续执行，结果在任务详情与对话中可见。

## 五、非功能与约束

- **性能**：对话首字响应 &lt; 2s（依赖网络与服务端）；列表与发现页支持分页或懒加载。
- **安全**：所有请求经 Gateway 鉴权；敏感操作需审批；支持内网/离线部署。
- **兼容**：桌面支持 Windows/macOS；Web 支持现代浏览器；嵌入式与 Q-EDA 宿主一致。
