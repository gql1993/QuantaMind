# QuantaMind 多智能体协作研发流程

本目录定义基于**多智能体协作**的研发流程：架构师、研发工程师、测试工程师三个智能体（或角色）按阶段协作，基于《QuantaMind量智大脑详细设计说明书》与《QuantaMind产品原型说明》交付可运行产品。

## 一、智能体角色

| 智能体 | 职责 | 产出物 |
|--------|------|--------|
| **架构师（Architect）** | 技术方案、接口契约、模块边界、技术选型与演进决策 | 架构决策记录（ADR）、接口规范、模块划分说明 |
| **研发工程师（Developer）** | 按架构与产品原型实现代码，遵循详细设计 | 代码、API 实现、客户端页面、配置与脚本 |
| **测试工程师（Tester）** | 测试策略、用例、自动化测试、质量门禁 | 测试计划、用例列表、自动化测试代码、报告 |

## 二、协作流程

```
  需求/详细设计 + 产品原型
           │
           ▼
  ┌─────────────────────────────────────────────────────────┐
  │ 阶段 N：架构师 → 产出架构决策与接口/模块说明              │
  │         研发工程师 → 按架构实现功能                      │
  │         测试工程师 → 编写用例与自动化，执行验证           │
  └─────────────────────────────────────────────────────────┘
           │
           ▼
  评审/合并 → 进入下一阶段 或 发布
```

- **输入**：详细设计说明书、产品原型说明、原型 HTML；上一阶段产出（ADR、接口变更、已知缺陷）。
- **输出**：可运行/可测的增量；测试报告；更新后的文档与代码库。

## 三、智能体指令与使用方式

- **架构师**：使用 `agents/architect.md` 作为该智能体的系统指令/角色定义；在 Cursor 或协作平台中创建“架构师”会话，将设计说明书与当前阶段目标注入，由其产出 ADR 与接口说明。
- **研发工程师**：使用 `agents/developer.md`；在“研发工程师”会话中注入架构师产出与产品原型，由其实现代码并提交。
- **测试工程师**：使用 `agents/tester.md`；在“测试工程师”会话中注入接口与实现说明，由其编写测试并执行，产出报告。

可**串行**（先架构师 → 再研发 → 再测试）或**并行**（架构师定接口后，研发与测试并行：研发实现、测试写用例并随实现完成后执行）。

## 四、研发路线图

见 **ROADMAP.md**：按阶段列出各智能体任务、交付物与验收标准；与详细设计说明书中的实施阶段（§13.4）对齐。

## 五、目录结构

```
dev/
├── README.md                 # 本文件：多智能体协作流程说明
├── ROADMAP.md                # 分阶段研发路线图与任务分配
├── agents/
│   ├── architect.md         # 架构师智能体角色与指令
│   ├── developer.md         # 研发工程师智能体角色与指令
│   └── tester.md            # 测试工程师智能体角色与指令
├── adr/                      # 架构决策记录（由架构师产出）
└── (仓库根 tests/)           # Gateway/API 自动化测试在仓库根 tests/（由测试工程师产出）
```

- 产品与界面依据：`docs/QuantaMind产品原型说明.md`、`prototype/index.html`
